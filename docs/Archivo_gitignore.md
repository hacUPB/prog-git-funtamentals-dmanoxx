# Archivo .gitignore

## ¿Qué es un archivo .gitignore?

El archivo `.gitignore` es un archivo de configuración utilizado por Git que permite indicar qué archivos o carpetas deben ser ignorados por el sistema de control de versiones.

Esto significa que Git no hará seguimiento de esos archivos, aunque existan dentro del proyecto.

## ¿Para qué se utiliza en un proyecto de programación?

El archivo `.gitignore` se utiliza para:

- Evitar subir archivos temporales o innecesarios.
- No versionar archivos generados automáticamente.
- Proteger información sensible como configuraciones locales o variables de entorno.
- Mantener el repositorio limpio y organizado.

## Importancia del archivo .gitignore

El uso de `.gitignore` es una buena práctica porque:
- Facilita el trabajo colaborativo.
- Reduce errores y conflictos.
- Mejora la seguridad del proyecto.
- Evita subir archivos que no son parte del código fuente.

## Uso del .gitignore en proyectos Python

En proyectos Python se suelen ignorar:
- Entornos virtuales
- Archivos compilados
- Archivos temporales
- Variables de entorno

## Conclusión

El archivo `.gitignore` es esencial en cualquier proyecto que utilice Git, ya que permite controlar qué archivos se comparten y cuáles no dentro del repositorio.
